历史

编年史

第一纪：Windows Live Spaces -> QQ空间 -> 网易博客

第二纪：主机屋 -> 西部数码 -> Openshift -> 000webhost

第三纪：github pages & coding静态网站

　

大事记

第一纪

最初接触的博客空间是微软的 Windows Live Spaces，花了很多时间去经营。然而，因为微软经营策略调整，空间被关闭了。

被迫转移到了 QQ空间，优点是QQ上的好友能带来很多流量。然而各种功能都需要充值，而且整体显得花里胡哨，随之放弃。

在 网易博客 和 新浪博客 之间摇摆思考良久，最终选择了 网易博客。使用期间，费尽心思经营博客，还自己修改主题。然而，随着政策的改变，审核变得非常玄学，文章经常被大面积封禁，而且不给理由。坚持数月后，还是放弃了。

　

第二纪

因为对博客空间的不信任，于是决定自建博客。首先，选择的是 主机屋 的免费空间，随后转为付费空间。但糟糕的宕机时有发生，很难维持博客的正常运行。然后，深刻的理解到便宜没好货，以及品牌的价值。

有了之前的教训，直接挑选了一个比较有名的主机提供商 西部数码。事实证明，这确实是一个很好的主机空间，价格适中，访问很快，基本没有发生宕机，性价比很高。而且，我通过 西部数码 给网站备案了。

随着工作生活变得繁琐麻烦，对于博客的维护渐渐忽视。某天，突然觉得花钱买主机空间很浪费，于是又转回了免费空间。因为对国外空间的迷之相信，这次选择了 Openshift 作为博客的主机空间。然而，好景不长，Openshift 技术策略调整，想要继续主机空间需要进行迁移，然而我无法达成迁移条件，只好继续转移。

000webhost 是一个非常老牌的国外免费主机空间提供商，口碑良好，于是迅速入驻。事实也是如此，风平浪静的经营许久，直到有一天主机空间被关闭了。后知后觉的我，在翻看邮箱时发现，原来是因为我的网站访问量渐渐变多，超过了他们设定的阈值，所以要我转成付费用户。套路很深，我选择逃跑。

　

第三纪

思考良久，四个月之后，我选择在 github pages 上重新搭建自己的博客，虽然需要一定的技术能力，但背靠微软的 github 还是给了我很大的信心。

偶然知道了“双线解析”，就是同一个域名，根据访问者的所属地不同，可以访问不同服务器的资源，这样会加快不同地区的访问速度。所以，又花了一些时间，将网站部署到了 coding静态网站 上。随后，设置域名的解析规则，国内访问者会解析到 coding静态网站，国外访问者则会解析到 github pages。

于是，现在就是新的开始了。
